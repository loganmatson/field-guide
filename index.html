Family Feud

Three voices argue any decision. A skeptic stress-tests it, an analyst weighs the base rates, and a systems-thinker maps the second-order effects. A host then names the real tradeoff none of them said out loud.

Why it works together · each voice is its own agent with its own job. Swapping the skeptic for a tougher one is a one-line change, because no single agent owns the whole debate.

Drift Auditor

Scans every skill and every memory file at the end of a session and checks each one against its stated purpose. Catches a renamed file a skill still points to, a convention that quietly broke, a memory that went stale.

Why it works together · it keeps all forty skills aligned so the work stays organized. Runs automatically when a session ends, so nothing has to be remembered by hand.

06 · The compounding math

Every skill multiplies the last.

Add a skill and it lifts everyone who can run it, then makes the next one cheaper to build. The first costs the most. By the tenth, the curve bends up on its own.

Headcount · linearSkills + memory · compounding

The first skill is the expensive one. You invent the conventions, the memory, the file shapes. The second reuses all of it. By the tenth, you're snapping together pieces that already exist.

The mechanism · shared memory + conventions mean every new skill starts from a richer substrate. That's compound interest, paid in time.
